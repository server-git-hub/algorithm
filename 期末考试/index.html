1、简述Unity中协程（Coroutine）的作用及其实现方式?
正确答案：
在Unity中协程是一种特殊的函数，可暂停执行并在特定条件满足后继续运行，其作用及实现方式如下：
作用：
1、异步处理任务：避免主线程阻塞（如加载资源、延时操作），提升程序响应性。
2、控制帧间逻辑：按帧间隔执行代码（如每帧移动角色），适配不同设备帧率。
实现方式：
1、定义协程函数：函数返回类型为IEnumerator，通过yield return语句暂停执行。
2、启动与停止协程：用StartCoroutine()启动，StopCoroutine()或StopAllCoroutines()停止。
3、配合特殊返回值控制流程：yield return null（下一帧继续）、yield return new WaitForSeconds(时间)（延时后继续）。
2、简述Prefab 的作用及在程序中如何创建和销毁？
正确答案：
Prefab（预制体）是 Unity 中一种可重复使用的游戏对象模板，它包含了游戏对象及其所有组件、属性和子对象的配置。
Prefab 的主要作用：
1、高效复用资源：只需创建一次预制体，即可在场景中多次实例化（如重复生成敌人、道具等），避免重复配置组件，提升开发效率。
2、支持动态生成：可在运行时通过代码实例化预制体。
3、通过 Instantiate() 方法动态生成预制体实例。
4、通过 Destroy() 方法销毁实例化的Prefab（预制体）对象。
3、简述Unity中MonoBehaviour的生命周期函数，写出5个关键函数及其作用。
正确答案：
1. Awake()：脚本加载时最先调用，用于初始化变量。
2. Start()：第一次帧更新前调用，常用于依赖其他组件的初始化。
3. Update()：每帧调用，处理实时逻辑（如输入检测）。
4. FixedUpdate()：固定时间间隔调用，适用于物理引擎计算。
5. OnDestroy()：对象销毁时调用，用于释放资源。

4、简述Unity中物理引擎（PhysX）的主要组件及其作用。
正确答案：
1. Rigidbody（刚体）：赋予对象物理属性（质量、重力），接收力的作用。
2. Collider（碰撞器）：定义对象碰撞形状（盒体、球体等），触发碰撞检测。
3. Character Controller（角色控制器）：封装人物移动逻辑，支持爬坡、碰撞检测。
4. Joint（关节）：连接多个刚体，模拟铰链、弹簧等物理连接效果。
5. Force（力）：通过AddForce()等函数对刚体施加力，实现物理运动。

5、简述Unity中UI系统（UGUI）的核心组件，写出5个主要组件。
正确答案：
1. Canvas（画布）：UI的根容器，控制渲染顺序和缩放模式。
2. Image（图像）：显示纹理、精灵（Sprite）或渐变背景。
3. Text（文本）：显示可编辑或静态文本，支持富文本格式。
4. Button（按钮）：交互控件，可绑定点击事件（OnClick）。
5. Layout Group（布局组）：自动排列子元素（如Horizontal/Vertical Layout Group）。

6、写出Unity中光照系统的5个核心组件及其作用。
正确答案：
1. Directional Light（平行光）：模拟太阳等远距离光源，影响全局光照方向。
2. Point Light（点光源）：从一点向四周发射光线，影响球形范围内的对象（如灯泡）。
3. Light Probe Group（光照探针组）：采样动态对象的光照信息，解决静态光照无法照亮移动物体的问题。
4. Lightmap（光照贴图）：烘焙静态场景的光照结果，减少运行时计算开销。
5. Cookie（光域网纹理）：为光源添加形状投影（如树叶阴影、窗户光斑）。

7、写出Unity中物理材质（Physic Material）的5个核心属性及其作用。
正确答案：
1. Dynamic Friction（动态摩擦力）：对象移动时受到的摩擦力大小（0-1，0为无摩擦）。
2. Static Friction（静态摩擦力）：对象静止时抵抗移动的力（如冰面接近0，水泥地接近1）。
3. Bounciness（弹性）：碰撞后反弹的程度（0为不反弹，1为完全弹性碰撞）。
4. Friction Combine（摩擦力组合模式）：多个碰撞体材质摩擦值的计算方式（如Average/Minimum）。
5. Bounce Combine（弹性组合模式）：碰撞后弹性的计算方式（如Maximum使物体更易反弹）。

8、 简述Unity中粒子系统（Particle System）的核心模块。
正确答案：
1. Shape（形状）：定义粒子发射的形状（如球体、锥体、网格表面）。
2. Emission（发射）：控制发射速率（每秒发射粒子数）和Burst（爆发式发射）。
3. Velocity Over Lifetime（生命周期速度）：随时间变化的粒子速度（如模拟烟雾上升减速）。
4. Color Over Lifetime（生命周期颜色）：粒子从生成到消失的颜色渐变（如火焰从黄到红）。
5. Renderer（渲染器）：设置粒子显示方式（Sprite/网格）、排序层级及接收阴影。

9、简述Unity中5种物理关节及其应用场景。
正确答案：
1. Hinge Joint（铰链关节）：模拟门、抽屉的旋转运动（限制单轴旋转）。
2. Spring Joint（弹簧关节）：模拟绳索、弹性物体（如弹弓、蹦床）。
3. Fixed Joint（固定关节）：将两个刚体固定在一起（如角色手持武器）。
4. Character Joint（角色关节）：模拟人体骨骼连接（限制多轴旋转范围）。
5. Configurable Joint（可配置关节）：自定义自由度和力反馈（如机械臂、复杂机械结构）。

10、写出Shader中5种常用纹理类型及其作用。
正确答案：
1. Albedo Map（反照率纹理）：控制物体基础颜色（如皮肤、金属的本色）。
2. Normal Map（法线纹理）：通过RGB值存储表面法线，模拟细节凹凸（如岩石纹理）。
3. Metallic Map（金属度纹理）：控制材质金属属性（0为非金属，1为金属）。
4. Roughness Map（粗糙度纹理）：控制表面粗糙程度（影响反射模糊）。
5. Emission Map（自发光纹理）：定义物体发光区域（如屏幕、霓虹灯）。

11、写出游戏开发场景中，常用碰撞器的类型及其作用。
正确答案：
1. 盒体碰撞器（Box Collider）：呈立方体形状，用于模拟规则几何形体的碰撞，如箱子、墙壁等，计算效率高。
2. 球体碰撞器（Sphere Collider）：呈球形，适用于需要球形碰撞判定的物体，如球类物体交互。
3. 胶囊体碰撞器（Capsule Collider）：类似胶囊形状，常用于角色模型的碰撞检测，贴合人体轮廓，兼顾效率与准确性。
4. 网格碰撞器（Mesh Collider）：可根据模型网格形状生成复杂碰撞形状，用于不规则物体（如地形、家具），但计算开销较大。
5. 车轮碰撞器（Wheel Collider）：专门用于模拟车辆车轮的物理碰撞，支持悬挂系统、摩擦力等物理属性设置，实现车辆行驶效果。

12、角色控制器的优点有哪些？
正确答案：
1. 无需刚体即可实现碰撞移动：直接封装了重力、斜坡检测和碰撞逻辑，避免因刚体物理计算导致的抖动或穿透问题，适合需要精准控制移动的角色。
2. 简化复杂地形交互：自动处理爬坡、台阶攀爬（通过设置Step Offset），无需额外编写碰撞检测代码，提升开发效率。
3. 性能开销更低：相比刚体+碰撞器的组合，角色控制器（Character Controller）的物理逻辑更轻量，适合移动端或大量角色同屏的场景。
4. 精准的移动控制：通过Move()方法直接控制位移，可精确控制角色在每一帧的移动距离，适合需要严格同步的多人游戏或动作游戏。
5. 内置重力与跳跃逻辑：无需手动计算重力加速度，直接通过SimpleMove()实现受重力影响的移动，跳跃时只需设置垂直速度即可。

13、3D 拾取技术中，射线检测的原理是什么？
正确答案：
1. 射线发射机制：从指定起点（如相机位置、鼠标点击坐标）沿特定方向发射一条无限细的射线，以向量数学形式定义起点和方向。
2. 碰撞检测逻辑：射线与场景中带有碰撞器（Collider）的物体进行相交计算，判断是否发生碰撞，仅检测开启碰撞功能的物体。
3. 交点信息获取：若碰撞发生，计算交点的世界坐标、局部坐标及表面法线，同时返回碰撞物体的引用（GameObject）和碰撞器组件（Collider）。
4. 层级与掩码过滤：通过Layer Mask指定射线仅检测特定层级的物体（如“Player”“Environment”），避免无效碰撞计算。
5. 距离与精度控制：可设置射线最大检测距离，结合物理材质（如摩擦、弹性）影响碰撞反馈，优化性能消耗。

14、Unity 光照系统中光探头的工作方式和类型有哪些？
正确答案：
1、光探头（Light Probe）用于捕捉场景中的间接光照，为动态物体提供光照信息。
2、工作时，在场景中布置光探头组，光探头会采集周围环境的光照数据。
3、当动态物体在场景中移动时，通过插值计算物体所在位置附近光探头的数据，从而为其提供连续、合理的光照效果，解决动态物体无法使用烘焙光照的问题。
分类：
1、烘焙光探头（Baked Light Probe）：预计算光照数据，适合静态场景，性能最优（如Unity的Light Probe Group）。
2、实时光探头（Realtime Light Probe）：动态更新光照数据，适用于光源变化场景（如灯光开关、昼夜循环），但消耗更多资源。

15、简述刚体组件中Is Kinematic属性的作用及与碰撞器的关系。
正确答案：
核心作用：控制刚体是否受物理引擎驱动。
1、开启（True）时：刚体不再受物理引擎的力（如重力、碰撞推力）影响，仅能通过脚本或动画直接控制位置和旋转。
2、关闭（False）时：刚体完全受物理引擎控制，遵循牛顿力学（如重力下落、碰撞反弹）。
与碰撞器的关系：
1、 即使开启Is Kinematic，刚体仍可与其他碰撞器发生碰撞，但碰撞时不会被物理力推动，而是由脚本决定是否响应碰撞（如触发事件）。
2、若需物体既参与碰撞又不受物理影响（如玩家无法推动的墙壁），可同时开启Is Kinematic和碰撞器。
